Execute SQL and Visualize Data with Apache Superset

2025-12-21 database

Apache Superset is an open-source data visualization tool created at Airbnb.

When you run it with Docker Compose, it starts up Superset along with Redis and PostgreSQL, and performs initial setup with superset db upgrade, superset fab create-admin, and superset init. When upgrading versions, migration is performed by running db upgrade & init.

$ git clone --depth=1 https://github.com/apache/superset.git
$ cd superset
$ git checkout tags/5.0.0
$ docker compose -f docker-compose-image-tag.yml up
# open http://localhost:9000 admin/admin

Superset is similar to Redash, where you write SQL to create charts and add them to dashboards, but it offers a much wider variety of chart types and can be extended with plugins. By installing libraries like pyathena and pydynamodb, you can connect to various databases.

Additionally, when creating charts, selecting axes and aggregation columns generates SQL, enabling Tableau-like usage. SQL written can be saved as Virtual Datasets and reused across multiple charts. I think it’s a great tool that combines the flexibility, portability, and tunability of SQL-based approaches with the low barrier to entry for chart creation.

Connect Tableau Desktop to Athena and handle tables with struct columns and date split into year, month, and day - sambaiz-net

SELECT quarter AS quarter,
    product_line AS product_line,
    sum(sales) AS "SUM(sales)"
FROM (
    /* Original SQL */
    SELECT *
    FROM public.cleaned_sales_data
    LIMIT 100000
) AS virtual_table
WHERE year = 2004
GROUP BY quarter, product_line
ORDER BY "SUM(sales)" DESC
LIMIT 100

You can also create filters on dashboards. Enabling cross-filtering allows you to filter across the entire dashboard.

Superset is built with Flask-AppBuilder (FAB) and supports RBAC and authentication via OAuth and LDAP. For example, users with the Gamma role who don’t have the all datasource access on all_datasource_access permission cannot access anything by default. Once permissions are granted, Datasets and dashboards become visible. Mapping between external provider groups and roles can be done with AUTH_ROLES_MAPPING.
