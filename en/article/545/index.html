Add Entra ID to AWS IAM identity provider and sign in with SAML

2025-11-03 awsauthazure

Entra ID supports authentication via SAML in addition to OIDC.

Authenticating with Google Accounts using Microsoft Entra ID - sambaiz-net

While you can create and configure an app from scratch, you can skip settings by selecting the AWS Single-Account Access app from the app gallery.

Enable SAML in the SSO settings. Note that RoleSessionName only accepts a-zA-Z_0-9+=,.@-, so map to other fields as needed.

Download the federation metadata XML.

<?xml version="1.0" encoding="utf-8"?>
<EntityDescriptor ID="_REDACTED" entityID="https://sts.windows.net/REDACTED-TENANT-ID/"
	xmlns="urn:oasis:names:tc:SAML:2.0:metadata">
	<RoleDescriptor xsi:type="fed:SecurityTokenServiceType" protocolSupportEnumeration="http://docs.oasis-open.org/wsfed/federation/200706"
		xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
		xmlns:fed="http://docs.oasis-open.org/wsfed/federation/200706">
		...
	</RoleDescriptor>
	<RoleDescriptor xsi:type="fed:ApplicationServiceType" protocolSupportEnumeration="http://docs.oasis-open.org/wsfed/federation/200706"
		xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
		xmlns:fed="http://docs.oasis-open.org/wsfed/federation/200706">
		...
	</RoleDescriptor>
	<IDPSSODescriptor protocolSupportEnumeration="urn:oasis:names:tc:SAML:2.0:protocol">
		<KeyDescriptor use="signing">
			<KeyInfo
				xmlns="http://www.w3.org/2000/09/xmldsig#">
				<X509Data>
					<X509Certificate>REDACTED-CERTIFICATE-1</X509Certificate>
				</X509Data>
			</KeyInfo>
		</KeyDescriptor>
		...
		<SingleLogoutService Binding="urn:oasis:names:tc:SAML:2.0:bindings:HTTP-Redirect" Location="https://login.microsoftonline.com/REDACTED-TENANT-ID/saml2" />
		<SingleSignOnService Binding="urn:oasis:names:tc:SAML:2.0:bindings:HTTP-Redirect" Location="https://login.microsoftonline.com/REDACTED-TENANT-ID/saml2" />
		<SingleSignOnService Binding="urn:oasis:names:tc:SAML:2.0:bindings:HTTP-POST" Location="https://login.microsoftonline.com/REDACTED-TENANT-ID/saml2" />
	</IDPSSODescriptor>
</EntityDescriptor>

Add this file as a metadata document to create an IAM Identity provider.

Create a Role that allows the Provider to AssumeRoleWithSAML.

{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Effect": "Allow",
            "Principal": {
                "Federated": "arn:aws:iam::(account):saml-provider/(provider_name)"
            },
            "Action": "sts:AssumeRoleWithSAML",
            "Condition": {
                "StringEquals": {
                    "SAML:aud": "https://signin.aws.amazon.com/saml"
                }
            }
        }
    ]
}

Finally, create and assign an Entra ID app role to users to return the value in the format arn:aws:iam::(account):role/(role_name),arn:aws:iam::(account):saml-provider/(provider_name) via the Role attribute, and you can sign in from https://myapps.microsoft.com/.

You can use saml2aws to obtain temporary credentials for the AWS CLI.

$ saml2aws configure \
  --idp-provider=Browser \
  --url=https://launcher.myapps.microsoft.com/api/signin/(app_id)?tenantId=(tenant_id) \
  --profile=entraid \
  --skip-prompt

$ cat ~/.saml2aws
$ saml2aws login --skip-prompt --download-browser-driver
$ aws sts get-caller-identity --profile entraid
{
    "UserId": "*****:(email)",
    "Account": "(account)",
    "Arn": "arn:aws:sts::(account):assumed-role/*****/(email)"
}
