Expose Celery Metrics in Prometheus Format with celery-exporter

2026-02-09 pythonprometheus

Celery is a distributed task queue for Python. Workers pull tasks from a Broker like Redis and execute them. It also provides features like retry, scheduling, and workflow.

import os
from celery import Celery, chain, group

app = Celery('tasks', broker=os.environ.get('CELERY_BROKER_URL', 'redis://localhost:6379/0'))

app.conf.update(
    worker_send_task_events=True,
    task_send_sent_event=True,
)

@app.task
def add(x, y):
    return x + y

# Retry
@app.task(bind=True, max_retries=3, default_retry_delay=60)
def fetch_url(self, url):
    try:
        return requests.get(url).text
    except Exception as e:
        self.retry(exc=e)

# Workflow: sequential execution
chain(add.s(1, 2), add.s(3))()  # (1+2) + 3 = 6

# Workflow: parallel execution
group(add.s(1, 2), add.s(3, 4))()  # [3, 7]

# Scheduling
app.conf.beat_schedule = {
    'add-every-minute': {
        'task': 'tasks.add',
        'schedule': 60.0,
        'args': (1, 2),
    },
}

By default, tasks are removed from the queue when the worker ACKs upon receipt, so tasks in progress are lost when a worker crashes. With task_acks_late=True, ACK happens after completion, so tasks are redelivered after a crash. However, the same task may be re-executed.

app.conf.update(
    task_acks_late=True,  # default: False
)

Tasks are prefetched for efficiency, but for long-running tasks, reducing worker_prefetch_multiplier helps distribute processing more evenly.

app.conf.update(
    worker_prefetch_multiplier=1,  # default: 4
)

Start Redis, Celery Worker, and celery-exporter.

# docker-compose.yml
services:
  redis:
    image: redis:8.4
    ports:
      - "6379:6379"

  worker:
    build: .
    command: celery -A tasks worker --loglevel=info
    environment:
      - CELERY_BROKER_URL=redis://redis:6379/0
    depends_on:
      - redis

  celery-exporter:
    image: danihodovic/celery-exporter:0.12.2
    command: --broker-url=redis://redis:6379/0
    ports:
      - "9808:9808"
    depends_on:
      - redis

FROM python:3.13-slim
WORKDIR /app
RUN pip install --no-cache-dir celery==5.6.2 redis
COPY tasks.py .

Start and execute a task.

$ docker compose up -d --build
$ docker compose exec worker python -c "from tasks import add; add.delay(4, 6)"

celery-exporter aggregates task execution status from Celery events and exposes them in Prometheus format. Metrics such as task sent, received, started, succeeded, failed counts, runtime, and queue length can be obtained.
