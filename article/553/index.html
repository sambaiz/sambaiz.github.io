Apache Superset で SQL を実行しデータを可視化する

2025-12-21 databasepython

Apache Superset は Airbnb で作られた OSS の BI ツール。

Docker Compose で動かすと Superset 本体と共に Redis と PostgreSQL が立ち上がり、 superset db upgrade, superset fab create-admin, superset init で初期設定が行われる。 バージョンを上げる際も db upgrade & init することで migrate される。

$ git clone --depth=1 https://github.com/apache/superset.git
$ cd superset
$ git checkout tags/5.0.0
$ docker compose -f docker-compose-image-tag.yml up
# open http://localhost:9000 admin/admin

Superset は SQL を書いてチャートを描きダッシュボードに載せる Redash のようなツールだが、グラフの種類がとても多く、プラグインによって拡張することもできる。 pyathena や pydynamodb といったライブラリをインストールすることで様々なデータベースと接続できる。

Redashでデータを可視化する - sambaiz-net

また、チャートの作成時に軸や集計カラムを選ぶと SQL が生成されるので Tableau のような使い方もできる。 書いた SQL を Virtual Dataset として保存しておくと複数のチャートで再利用することができる。 SQL をベースとした自由度の高さとポータビリティ、チューニングのしやすさと、チャート作成時の敷居の低さを両立した良いツールだと思う。

Tableau Desktop を Athena に接続して年月日に分かれた日時やstructを含むテーブルを扱う - sambaiz-net

SELECT quarter AS quarter,
    product_line AS product_line,
    sum(sales) AS "SUM(sales)"
FROM (
    /* Original SQL */
    SELECT *
    FROM public.cleaned_sales_data
    LIMIT 100000
) AS virtual_table
WHERE year = 2004
GROUP BY quarter, product_line
ORDER BY "SUM(sales)" DESC
LIMIT 100

ダッシュボードにもフィルタを作成できる。cross-filtering を有効にすると全体に対してフィルタをかけられる。

Superset は Flask-AppBuilder (FAB) で作られており OAuth や LDAP による認証をサポートしている。 SupersetSecurityManager の oauth_user_info を override し、

import jwt
from superset.security import SupersetSecurityManager

class CustomSsoSecurityManager(SupersetSecurityManager):
    def oauth_user_info(self, provider, response=None):
        if provider == 'cognito':
            me = self.appbuilder.sm.oauth_remotes[provider].get('oauth2/userInfo').json()

            id_token_decoded = jwt.decode(
                response['id_token'],
                options={"verify_signature": False}
            )
            role_keys = id_token_decoded.get('cognito:groups', [])

            return {
                'username': data.get('email', data.get('sub')),
                'email': data.get('email'),
                'first_name': data.get('given_name', ''),
                'last_name': data.get('family_name', ''),
                'role_keys': role_keys,
            }
        return {}

また、RBAC によって権限を管理できる。例えば all datasource access on all_datasource_access パーミッションが紐づいていない Gamma ロールのユーザーはデフォルトで SQL Lab は開けず何のデータにもアクセスできない。Dataset のパーミッションを付与すると対応するダッシュボードも表示される。role_keys と Superset のロールとのマッピングは AUTH_ROLES_MAPPING で行う。

AUTH_ROLES_MAPPING = {
    'superset-admins': ['Admin'],
    'superset-alpha': ['Alpha'],
}

SQL Lab に関してはデフォルトで Row Level Security が無効になっている。クエリの書き換えが必要で、有効してもバイパスされる可能性があるようだ。実行時にデータベースへの接続 URL を書き換える DB_CONNECTION_MUTATOR を用いればユーザーやロールごとにクレデンシャルを書き換えることでデータベース側で制御することができる。テーブルのアクセス権限もそちらでまとめて管理した方が他のクライアントとの一貫性が保てて良さそうだ。Cognito で認証している場合、ID トークンを Session に書いておけば ID Pool でクレデンシャルを取得するといったこともできる。

Cognito Identity Pool のタスク/ルールベースマッピングで User Pool のグループに対応するロールの一時クレデンシャルを発行する - sambaiz-net

    credentials = # implement get_credentials(session.get('cognito_id_token'))
    new_query = dict(uri.query) if uri.query else {}
    new_query['aws_session_token'] = credentials['aws_session_token']

    new_uri = URL.create(
        drivername=uri.drivername,
        username=credentials['aws_access_key_id'],
