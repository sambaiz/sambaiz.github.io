Prometheus を CDK でインストールして Recording rules で集計したデータを New Relic に Remote write することでデータ量を節約する

2024-05-23 awskubernetesnewrelicprometheus

Prometheus の Recording rules は PromQL で既存のメトリクスから新たなメトリクスを作ることができる機能。これによって集計したデータを送ることで生のデータを送るのと比べて New Relic へのデータ送信量を節約することができるが、newrelic-bundle に含まれる newrelic-prometheus-configurator の Prometheus は Agent mode で動くので Recording rules に対応していない。

CDKでEKSクラスタにnewrelic-bundleをインストールしてモニタリングする - sambaiz-net

そこで Helm で自前の Prometheus をインストールして Remote write する。Recording rules のメトリクス名は level:metric:operations の形式にする。

const extraScrapeConfigs = 
`- job_name: newrelic-pods
  honor_timestamps: true
  scrape_interval: 30s
  scrape_timeout: 10s
  scheme: http
  follow_redirects: true
  enable_http2: true
  metrics_path: /metrics
  kubernetes_sd_configs:
  - role: pod
  relabel_configs:
  - source_labels: [__meta_kubernetes_pod_annotation_newrelic_io_scrape]
    separator: ;
    regex: 'true'
    replacement: $1
    action: keep
  - source_labels: [__meta_kubernetes_pod_phase]
    separator: ;
    regex: 'Pending|Succeeded|Failed|Completed'
    replacement: $1
    action: drop`

const recordingRules = {
  groups: [
    {
      name: 'example',
      interval: '5m',
      rules: [
        {
          record: 'code:prometheus_http_requests_total:sum',
          expr: 'sum by (code) (prometheus_http_requests_total)',
          labels: { aaaa: 'bbbb' },
        },
      ],
    },
  ],
}

const remoteWrite = [
  {
    url: 'https://metric-api.newrelic.com/prometheus/v1/write?prometheus_server=prometheus',
    authorization: {
      credentials_file: '/etc/newrelic-secrets/license-key',
    },
    write_relabel_configs: [
      {
        source_labels: ['aaaa'],
        separator: ';',
        regex: 'bbbb',
        action: 'keep',
      },
    ]
  }
]

const prometheus = cluster.addHelmChart('PrometheusChart', {
  release: 'prometheus',
  chart: 'prometheus',
  repository: 'https://prometheus-community.github.io/helm-charts',
  version: '25.21.0',
  namespace: 'prometheus',
  createNamespace: true,
  values: {
    alertmanager: {
      enabled: false,
    },
    'kube-state-metrics': {
      enabled: false,
    },
    'prometheus-node-exporter': {
      enabled: false,
    },
    'prometheus-pushgateway': {
      enabled: false,
    },
    extraScrapeConfigs,
    serverFiles: {
      'recording_rules.yml': recordingRules,
    },
    server: {
      remoteWrite,
      extraSecretMounts: [
        {
          name: 'newrelic-secret',
          secretName,
          mountPath: '/etc/newrelic-secrets',
          readOnly: true,
        },
      ],
    }
  },
})

この Chart には Prometheus server のほかに Alertmanager や Pushgateway なども含まれているが、今回は使わないので無効化している。

Prometheus server にポートフォワードして PromQL で Recording rules のメトリクスを取得してみる。

$ kubectl -n prometheus port-forward svc/prometheus-server 9090:80
Forwarding from 127.0.0.1:9090 -> 9090
Forwarding from [::1]:9090 -> 9090

New Relic にも送られていることが確認できる。

FROM Metric 
SELECT latest(`code:prometheus_http_requests_total:sum`) 
WHERE prometheus_server = 'prometheus' FACET code
